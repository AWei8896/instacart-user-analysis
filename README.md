# 生鲜电商用户复购行为与品类运营策略分析

> 基于 Instacart Market Basket Analysis 数据集 (300万+ 订单 / 20万+ 用户)

---

## 项目简介

本项目模拟**电商数据分析师**的真实工作场景，对大型生鲜电商平台的用户行为数据进行端到端分析。核心解决的问题是：**什么样的商品和用户行为能驱动复购？如何基于数据制定精准的品类运营策略？**

## 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| 数据处理 | **SQL (MySQL)** | 复购间隔分析、品类排名、用户分层 — 6 种窗口函数综合应用 |
| 分析与特征工程 | **Python (Pandas / NumPy)** | 数据清洗、多表合并、RFM 建模、关联分析 |
| 统计分析 | **SciPy / NumPy** | 分位数打分、Cohort 矩阵计算 |
| 数据可视化 | **Matplotlib + Seaborn** | 7 张分析图表 (热力图、雷达图、饼图、柱状图) |
| 版本管理 | **Git / GitHub** | 代码托管与项目展示 |

## 核心业务指标

| 指标 | 计算方式 | 业务含义 |
|------|----------|----------|
| **复购率** | 复购商品数 / 总购买商品数 | 衡量单品用户粘性 |
| **品类连带率** | P(买B `|` 已买A) | 识别交叉销售机会 |
| **LTV分层** | NTILE(5) R+F+M 总分 | 划分高/低价值用户 |
| **周留存率** | 首购后第 N 周仍购买的用户比例 | 衡量长期留存健康度 |
| **复购周期** | 用户相邻两单间隔天数的分布 | 指导推送时机 |

## SQL 窗口函数清单

本项目中使用的窗口函数全部写在 [sql/02_analysis_queries.sql](sql/02_analysis_queries.sql)：

- `LAG()` — 计算用户两次购买间隔
- `RANK()` / `NTILE()` — 品类排名、用户分桶
- `SUM() OVER()` — 计算占比的累计分母
- `AVG() OVER(ROWS BETWEEN ...)` — 购物篮大小的滚动平均
- `ROW_NUMBER()` — 商品复购率排序

## 可视化图表

| # | 图表 | 回答的问题 |
|---|------|-----------|
| 1 | 复购周期分布直方图 | 用户平均几天回购一次？ |
| 2 | Cohort 留存热力图 | 第 N 周获取的用户后续各周留存如何？ |
| 3 | RFM 分层占比饼图 | 高/低价值用户各占多少？ |
| 4 | RFM 雷达图 | 各层用户在活跃度/频率/消费力上的差异？ |
| 5 | 品类复购率 Top 15 柱状图 | 哪些品类最能留住用户？ |
| 6 | 品类连带率热力图 | 买牛奶的用户还买了什么？ |
| 7 | 订单时段热力图 | 星期几什么时段订单量最高？ |
A: 因为购买次数和金额通常是长尾分布，直接用原始值会被少数重度用户主导。用分位数法能保证各层用户数相对均衡，分层标签在运营上更有可操作性。

**Q: 这个分析对业务有什么帮助？**  
A: 三个直接可落地的方向：一是针对高复购品类做捆绑促销 (基于连带率)；二是针对中低价值用户做新品类引导 (基于RFM分层)；三是根据复购周期分布确定推送时机 (比如用户在购买后第5-7天触发优惠券推送)。
